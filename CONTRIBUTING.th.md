# การร่วมสมทบ (Contributing)

ขอบคุณที่สนใจร่วมพัฒนาและสมทบให้กับระบบ Lord of the Mysteries TTRPG

เรายินดีรับการสมทบทุกขนาด ไม่ว่าจะเป็นการรายงานบั๊ก การปรับปรุงเอกสาร การแก้ไขบั๊ก หรือฟีเจอร์ใหม่ๆ

## ก่อนที่คุณจะเริ่ม

หากคุณกำลังวางแผนทำฟีเจอร์ขนาดใหญ่หรือการเปลี่ยนแปลงที่สำคัญ กรุณาเปิด Issue ก่อน เพื่อให้เราได้พูดคุยเกี่ยวกับการออกแบบก่อนเริ่มการพัฒนา

หากคุณต้องการทำงานใน Issue ที่มีอยู่แล้ว ให้แสดงความคิดเห็นและขอให้เรามอบหมายงานนั้นให้คุณ เพื่อช่วยป้องกันการทำงานซ้ำซ้อน

## การตั้งค่าเพื่อการพัฒนา

ไฟล์ซอร์สโค้ดสำหรับระบบนี้อยู่ในไดเรกทอรีหลักและไดเรกทอรีย่อยโดยตรง

### ข้อกำหนด

- [Node.js](https://nodejs.org/) (ทางเลือก ขึ้นอยู่กับว่าคุณเขียนสคริปต์ build กำหนดเองหรือไม่)
- ติดตั้ง Foundry VTT ในเครื่องสำหรับการทดสอบ

### การติดตั้งบิลด์สำหรับการพัฒนา

สำหรับการพัฒนาอย่างต่อเนื่อง ให้ใช้ symlink หรือคัดลอกโฟลเดอร์ repository ไปยังไดเรกทอรีข้อมูลผู้ใช้ Foundry (Foundry User Data directory) ของคุณ

ตัวอย่าง (Linux/macOS):
```bash
ln -s /path/to/Lord-of-the-Mysteries-TTRPG $FoundryUserDataPath/systems/lotm
```
*(แทนที่ `$FoundryUserDataPath` ด้วยพาธไปยังไดเรกทอรีข้อมูลผู้ใช้ Foundry ของคุณ เช่น `~/.local/share/FoundryVTT/Data`)*

ตัวอย่าง (Windows - รันในฐานะ Administrator):
```cmd
mklink /D "C:\Users\<you>\AppData\Local\FoundryVTT\Data\systems\lotm" "C:\path\to\Lord-of-the-Mysteries-TTRPG"
```

## Pull Requests

กรุณา:

1. สร้าง feature fork & branch แทนที่จะ commit โดยตรงไปยัง main
2. ให้ pull requests มุ่งเน้นไปที่ฟีเจอร์เดียวหรือการแก้ไขเดียว
3. อ้างอิง issue ที่เกี่ยวข้องในคำอธิบาย PR (ตัวอย่างเช่น `Closes #42`)
4. ทดสอบการเปลี่ยนแปลงของคุณใน Foundry VTT ก่อนส่ง
5. อัปเดตเอกสารประกอบ (เช่น `README.md` และ `README.th.md`) หากการเปลี่ยนแปลงของคุณปรับเปลี่ยนพฤติกรรมของระบบอย่างเห็นได้ชัด

Pull requests อาจได้รับคำติชมสำหรับการตรวจสอบก่อนที่จะถูกรวม (merged)

## สไตล์การเขียนโค้ด

ไม่มีข้อกำหนดเรื่องการจัดรูปแบบที่เข้มงวด นอกเหนือจากการรักษาสไตล์ที่มีอยู่ให้สม่ำเสมอ

เมื่อเป็นไปได้:

- เน้นโค้ดที่ชัดเจน อ่านง่าย และดูแลรักษาง่าย มากกว่าวิธีแก้ปัญหาที่ซับซ้อน
- ให้ commits มุ่งเน้นและจัดกลุ่มอย่างมีตรรกะ
- หลีกเลี่ยงการเปลี่ยนแปลงการจัดรูปแบบที่ไม่เกี่ยวข้องใน pull request เดียวกัน

## การรายงานบั๊ก

เมื่อรายงานบั๊ก กรุณาระบุ:

- เวอร์ชันของ Foundry VTT
- เวอร์ชันของระบบ
- เบราว์เซอร์ (ถ้ามี)
- ขั้นตอนในการทำซ้ำปัญหา
- พฤติกรรมที่คาดหวัง
- พฤติกรรมที่เกิดขึ้นจริง
- ข้อผิดพลาดในคอนโซล (กด F12) หรือภาพหน้าจอ

ยิ่งให้ข้อมูลมากเท่าไหร่ ก็ยิ่งทำให้สามารถทำซ้ำและแก้ไขปัญหาได้ง่ายขึ้นเท่านั้น

## จรรยาบรรณ (Code of Conduct)

กรุณาแสดงความเคารพและให้ความร่วมมือเชิงสร้างสรรค์เมื่อโต้ตอบกับผู้ร่วมสมทบคนอื่นๆ เป้าหมายคือการสร้างโปรเจกต์ที่ยินดีต้อนรับทุกคน
